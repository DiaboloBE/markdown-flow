# 快速上手

学会编写 AI 指令来创建个性化内容。5分钟内，您将构建一个能适应每位访客的智能个人介绍。

**🌐 [在体验台实时试用](https://play.mdflow.run)** - 复制每个示例并测试！

**我们要构建什么：** 我们将从简单的**内容提示词**开始 - 这些指令测试 AI 是否理解您的意图并具有相关知识和能力。然后我们会逐步让它们变得更复杂和个性化。

## 第1步：基本 AI 指令（1分钟）

不要编写固定内容，而要为 AI 编写指令：

```markdown
为 Alex Chen 生成个人介绍。

包含以下事实：
- 从事科技行业
- 5年工作经验
- 构建应用程序和网站
- 喜欢徒步旅行和摄影
- 始终学习新技术

保持专业和友好。
```

**🚀 现在就在体验台试试！** 您会看到 AI 生成完整的介绍。

但有一个问题 - 每个人都得到相同的通用结果。

## 第2步：添加个性化（1分钟）

让我们让它适应不同的访客：

```markdown
询问访客的身份。客户、招聘官、同事、朋友，还是潜在约会对象？

?[%{{visitor_type}}潜在客户|招聘官|同事|朋友|潜在约会对象]

为{{visitor_type}}量身定制生成 Alex Chen 的介绍。

包含以下事实：
- 从事科技行业  
- 5年工作经验
- 构建应用程序和网站
- 喜欢徒步旅行和摄影
- 始终学习新技术

针对{{visitor_type}}调整语气和重点。
```

**🚀 在体验台测试这个！** 尝试不同的访客类型 - 看看相同的事实如何创建完全不同的介绍。

## 第3步：添加详细指令（2分钟）

让您的指令更具体以获得更好、更一致的结果：

```markdown
询问访客的身份。客户、招聘官、同事、朋友，还是潜在约会对象？

?[%{{visitor_type}}潜在客户|招聘官|同事|朋友|潜在约会对象]

为{{visitor_type}}量身定制生成 Alex Chen 的介绍：

- 对于潜在客户：强调可靠性、成功项目和解决问题的技能
- 对于招聘官：突出技术技能、职业成就和专业成长  
- 对于同事：关注协作、共同兴趣和团队合作
- 对于朋友：分享个人兴趣、爱好和生活乐趣
- 对于潜在约会对象：展示个性、生活方式和让关系愉快的特质

包含这些具体事实：
- 在科技领域工作5年
- 构建应用程序和网站
- 喜欢徒步旅行和摄影
- 始终学习新技术
```

**🚀 在体验台测试！** 将此与第2步比较 - 注意更详细的指令如何创建更有针对性、更相关的介绍。

**重要提示：** 您提供的具体事实越多，AI "幻觉"（编造信息）就越少。始终为 AI 提供真实细节以供使用。

## 第4步：防止 AI 幻觉（1分钟）

AI 有时会在没有足够细节时编造信息。让我们修复这个问题：

```markdown
询问访客的身份。客户、招聘官、同事、朋友，还是潜在约会对象？

?[%{{visitor_type}}潜在客户|招聘官|同事|朋友|潜在约会对象]

向{{visitor_type}}问好，然后介绍自己。您必须仅为{{visitor_type}}定制您的个人信息。遵循以下规则：

- 对于潜在客户：强调可靠性、成功项目和解决问题的技能
- 对于招聘官：突出技术技能、职业成就和专业成长  
- 对于同事：关注协作、共同兴趣和团队合作
- 对于朋友：分享个人兴趣、爱好和生活乐趣
- 对于潜在约会对象：展示个性、生活方式和让关系愉快的特质

仅使用以下关于 Alex 的已验证事实：
- 年龄：28岁
- 位置：旧金山
- 工作：TechCorp 高级软件工程师
- 经验：网页开发5年经验
- 技能：React、Node.js、Python
- 项目：领导团队构建了被5万+用户使用的电商平台
- 爱好：在马林县周末徒步、风景摄影
- 教育：加州大学伯克利分校计算机科学学位
- 个人：喜欢尝试新咖啡店，有一只名叫 Luna 的救援犬

不要编造额外细节。保持真实且引人入胜。
```

**🚀 在体验台试试这个！** 注意提供具体、详细的事实如何在防止 AI 编造信息的同时仍能创建个性化内容。

## 第5步：添加文档提示词（1分钟）

到目前为止，我们一直在编写**内容提示词**。如果您在体验台测试过，您可能会注意到 AI 有时使用不一致的语气，从错误的角度写作，使用不合适的格式，或者不太符合预期受众的感觉。

**文档提示词**通过设置 AI 的角色和全局行为来解决这些问题：

**文档提示词**：

```markdown
您是 Alex Chen。您的任务：
- 编写一个自然、真实的自我介绍，直接面向{{visitor_type}}。
- 使用第一人称"我"和"我的"。
- 介绍应该温暖、真诚，准备好显示在您的个人主页上而无需任何编辑。
- 保持对话和亲切的语气，就像您真的在见这个人一样。
- 不要包含任何元评论、指令或格式说明 - 只有 Alex 会说的实际介绍文本。
```

**内容提示词**（我们一直在构建的）：

```markdown
询问访客的身份。客户、招聘官、同事、朋友，还是潜在约会对象？

仅使用以下关于 Alex 的已验证事实：
```
